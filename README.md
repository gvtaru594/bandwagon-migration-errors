## 一、先看你拿到的是哪条报错

不同报错，原因差八万里。先对照一下：

| 错误提示 | 错误码 | 大致含义 |
|---|---|---|
| Migration backend is currently not available for this VPS. | 734152 | **IP 被封**，或后端临时故障 |
| Migration system is currently not available. | 734151 | 迁移系统临时故障，稍等就好 |
| Region is full. Please try again in a few hours. | 734104 | 目标机房满了，等几小时再试 |
| New location has no available IP addresses. | 734105 | 目标机房 IP 资源耗尽，等待补仓 |
| Failed to migrate | 734205 | 目标机房对你的方案不可用 |
| KiwiVM control panel is temporarily unavailable. | 3009005 | 面板维护中，稍后即恢复 |

报错如果不是 734152，基本都属于"临时性问题"——等等就好，稍后重试大概率能过。

**最棘手的是 734152**。这条提示看起来说的是"后端临时不可用"，实际上绝大多数情况是：**你的 IP 已经被封了**，所以系统拒绝让你迁移。

## 二、734152：十有八九是 IP 被封了

这是搬瓦工最常见的换机房失败情形。

搬瓦工换机房本质上是把整台 VPS 迁移走，走之前后端会做一次 IP 连通性检查。你的 IP 如果通不过这个检查，迁移命令就不会下达——所以你看到的是"后端不可用"，但真正的原因是 IP 已经被防火墙屏蔽了。

**怎么确认 IP 是不是真的被封了？**

进 KiwiVM 后台，在首页找到你的 VM ID（页面顶端那串数字）。然后在浏览器地址栏里访问这个 URL（把 YOUR_VMID 替换成你自己的数字）：


https://kiwivm.64clouds.com/YOUR_VMID/main-exec.php?mode=blacklistcheck


如果结果显示 "IP address has been blocked"，那就确认了：IP 封了，这才是根本原因。

**IP 被封后能不能靠换机房来换 IP？**

不能。这是个很容易踩的误区。搬瓦工系统设计上就不允许用换机房的方式绕过 IP 被封的问题——检测不通过，迁移根本启动不了。

解法只有一个：**先换 IP，再换机房**。

搬瓦工目前支持在 KiwiVM 面板申请更换 IP，具体操作在 KiwiVM 面板的 "IP Addresses" 部分。换完 IP、确认新 IP 可用之后，再去操作换机房就不会卡 734152 了。

## 三、734104 / 734105：目标机房满了，等就行

这两条都属于"目标机房暂时没资源"。

- 734104：机房整体容量满了
- 734105：机房 IP 地址库存耗尽，没法给你分配新 IP

用过搬瓦工一段时间的人对这个都不陌生。遇到某个机房（比如 DC9 CN2 GIA）很多人同时往里迁，资源就会短暂告急。

解法很简单：等。过几个小时，甚至十几分钟后再试，通常就能过。

不用一直盯着，可以每隔一两个小时点一次试试。如果等了一整天还不行，换一个目标机房先用着，等那个机房补仓了再迁回来。

这两个基本不用太担心。

## 六、换机房成功后连不上，这也是常见问题

说句题外话——有时候迁移本身是成功的，收到了完成邮件，但 SSH 就是连不上。这通常有两种原因：

1. **网络初始化还没完成**：迁移后新机房需要几分钟初始化网络配置，可以登 KiwiVM 里的 VNC（Root shell），直接看 VPS 状态。屏幕上如果显示 "A start job is running for Raise network interfaces"，说明还在初始化，等它跑完就好。

2. **IP 没有自动更新**：迁移后 IP 地址会变，如果你的 VPS 里还留着旧机房的网络配置，就会出问题。同样可以通过 KiwiVM 的 VNC 进去查一下网络配置，手动更新 IP 信息。

## 常见问题解答

**Q：换机房会不会丢数据？**  
A：不会。搬瓦工迁移机房会把整台 VPS 完整迁过去，数据、系统配置都会保留。迁移期间 VPS 会关机，迁完重启就能正常用。

**Q：换机房要钱吗？**  
A：不要。搬瓦工换机房是免费功能，没有次数限制，可以随时切换。

**Q：换机房后流量怎么算？**  
A：取决于目标机房。KVM 套餐迁入部分 CN2 机房后流量会按比例缩减（具体比例看目标机房规则），迁入普通机房的话流量不变。CN2 GIA-E 套餐在各机房间切换不影响总流量额度。

**Q：一直报 734152 但 IP 没有被封，该怎么办？**  
A：先确认 IP 状态（用 KiwiVM 的 blacklistcheck 工具），如果检测结果正常但仍无法迁移，可以提交工单联系搬瓦工客服，说明 VM ID 和错误码，客服会手动介入处理。

**Q：KVM 套餐能迁到 CN2 GIA 机房吗？**  
A：不行。CN2 GIA-E 和 CN2 GIA 机房是 CN2 GIA-E 套餐专属，KVM 常规套餐只能在普通机房之间切换。要用 CN2 GIA 机房，需要购买对应套餐。
